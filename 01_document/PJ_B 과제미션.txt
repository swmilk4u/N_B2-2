[Project B] 뉴스 요약 자동화 워크플로우 만들기⁠⁠⁠⁠⁠⁠⁠

1. 미션 소개
새로운 프로젝트를 시작할 때 가장 먼저 하는 일은 시장 조사입니다.
하지만 경쟁사의 동향을 파악하거나 고객의 생생한 반응을 확인하기 위해 수십 개의 커뮤니티와 블로그, 뉴스 사이트를 일일이 뒤지다 보면 금세 지치기 마련입니다.
수백 개의 탭을 띄워놓고 내용을 복사해서 엑셀에 옮기다 보면, 정작 중요한 ‘분석’은 시작도 하기 전에 이미 에너지는 바닥나고 말죠.
"수집부터 분석까지 자동으로 돌아가면 좋겠다"는 생각은 해봤지만 각 단계를 하나의 서비스로 연결하는 건 막상 어디서부터 시작해야 할지 막막했을 것입니다.
RSS 수집에서 AI 요약을 거쳐 협업 도구에 자동 저장하는 이 흐름을 코드 없이 직접 설계하는 것이 이번 프로젝트입니다.
외부 데이터 소스인 RSS 피드로 최신 기술 뉴스를 자동 수집하고 생성형 AI 툴로 내용을 요약하며 노션 등 협업 도구에 자동 저장하는 전체 워크플로우를 구축합니다.
단순히 하나의 기능을 구현하는 것이 아니라 스케줄 트리거와 주제 필터링 그리고 AI 요약과 에러 처리까지 자동화 파이프라인의 전체 흐름을 설계하고 연결하는 것이 핵심입니다.
이 과정을 마치면 파이프라인이 실제로 돌아가는 걸 눈으로 확인하는 순간 자동화가 어떤 의미인지 체감하게 될 것입니다.
다양한 AI 툴과 자동화 서비스를 유기적으로 연결하는 경험은 이후 더 복잡한 업무 자동화 시스템을 설계하고 운영하는 데 탄탄한 기반이 됩니다.

2. 최종 결과물
다음 3가지 산출물을 포함한 뉴스 요약 자동화 시스템을 팀 단위로 완성한다.
2.1 자동화 워크플로우
 - 매일 설정된 시간에 자동 실행되는 워크플로우
 - 워크플로우 구조를 보여주는 스크린샷
 - 각 단계별 역할과 연결 구조를 설명하는 문서
2.3 노션 데이터베이스 페이지
 - AI가 요약한 텍스트가 포함된 뉴스 콘텐츠
 - 제목, 요약문, 원문 링크, 발행일시가 각각 올바른 속성에 저장된 상태
2.3 README(또는 기획서)
 - 팀 역할/개인별 작업 요약
 - 주제 필터링 기준(키워드/태그 목록)과 선택 이유
 - 에러 처리 정책과 선택 이유

3. 과제 목표
이 과제를 마친 후, 학습자는 아래를 스스로 설명할 수 있어야 한다.
 - RSS 피드와 자동화 툴을 활용하여 데이터 수집 → AI 가공 → 저장으로 이어지는 자동화 워크플로우를 설명할 수 있다.
 - 생성형 AI 툴을 활용하여 텍스트 요약을 자동화하는 방법을 설명할 수 있다.
 - 스케줄 트리거와 에러 처리를 포함한 안정적인 자동화 시스템의 설계 원리를 설명할 수 있다.

4. 기능 요구 사항
다음 요구사항을 모두 만족해야 한다.
4.1데이터 수집
 - RSS 피드를 통해 기술 뉴스를 자동으로 수집한다.
 - 매일 설정된 시간에 트리거가 자동으로 작동해야 한다.
 - 수집된 뉴스 중 사전에 정의된 주제(예: AI)에 해당하는 1건을 선택하여 제목, 링크, 본문 내용을 추출한다.
4.2 AI 가공
 - 생성형 AI 툴을 활용하여 뉴스 내용을 3줄 이내로 요약한다.
4.3 자동 저장
 - 노션 등 협업 도구의 데이터베이스에 결과물을 자동 저장한다.
 - 제목, 요약문, 원문 링크, 발행 일시가 각각 올바른 속성에 저장되어야 한다.
4.4 안정성
 - 전체 프로세스가 사람의 개입 없이 자동으로 완료되어야 한다.
 - 뉴스가 수집되지 않거나 처리 중 오류가 발생한 경우 적절히 처리되어야 한다.
4.5 비용/중복 방지
 - 동일 기사 중복 저장 방지 키 1개 이상 적용을 권장한다(예: 원문 링크 해시, GUID).
 - 요약 호출은 기사 1건당 1회 원칙(재시도 제외)으로 한다.
 - 재시도는 최대 2회로 상한을 유지한다.

5. 보너스 과제 (선택)
 - 보너스 1 – 썸네일 이미지 자동 생성
이미지 생성 AI 툴을 활용하여 뉴스 주제에 맞는 썸네일 이미지를 생성하고 함께 저장한다.
 - 보너스 2 – 감성 분석 태그 추가
뉴스 내용의 긍정/부정 감성을 분석하여 태그로 분류한다.

6. 제약 사항
 6.1 도구
 - 자동화 툴은 Make, Zapier, n8n 등 자유롭게 선택하되, 사용한 툴의 이름을 명시한다.
 - 생성형 AI 연동 시 OpenAI API 또는 이에 준하는 모델을 사용한다.
 - 저장 도구는 노션을 권장하며, 다른 도구 사용 시 명시한다.
 6.2 금지 사항
 - RSS 피드가 없는 사이트를 강제로 크롤링하여 저작권을 침해하는 방식
 - 무한 루프에 빠지는 트리거 설정 (불필요한 API 호출 비용 발생 주의)
 6.3 보안
 - API 키/토큰은 환경 변수 또는 비공개 저장소에 보관한다.
 - 문서/스크린샷에 API 키/토큰이 노출되지 않도록 마스킹 처리한다.

# 워크플로우 설명서 (Workflow Design)

대상 파일: [`workflow/news-summary.n8n.json`](../workflow/news-summary.n8n.json)
자동화 툴: **n8n** · AI: **OpenAI gpt-4o-mini** · 저장소: **Notion**

---

## 1. 전체 구조도

```
[스케줄 트리거(매일 09:00, Asia/Seoul)]
        ↓
[RSS 수집(TechCrunch)]
        ↓
[주제 필터(AI 키워드 → 최신 1건)]
        ↓
[뉴스 있음?] ──false──▶ [알림: 뉴스 없음]   (정상 종료)
        │ true
        ▼
[Notion 중복 조회(원문 링크 키)]
        ↓
[신규 기사?] ──false──▶ [중복: 스킵]         (저장·요약 생략)
        │ true
        ▼
[OpenAI 요약(3줄 + 감성)]
        ↓
[요약 파싱]
        ↓
[Notion 저장(제목/요약/링크/발행일시…)]
        ↓
[완료]

[에러 트리거] ──▶ [알림: 처리 실패]          (별도 경로, 실패 시 발화)
```

---

## 2. 노드별 역할

| #   | 노드                      | 타입                    | 역할                                                         | 핵심 설정                                            |
| --- | ----------------------- | --------------------- | ---------------------------------------------------------- | ------------------------------------------------ |
| 1   | **스케줄 트리거 (매일 09:00)**  | Schedule Trigger      | 매일 정해진 시각에 파이프라인을 자동 시작                                    | `days` 간격, 09:00 / 워크플로우 타임존 `Asia/Seoul`        |
| 2   | **RSS 수집 (TechCrunch)** | RSS Feed Read         | 지정 RSS에서 최신 기사 목록을 가져옴                                     | `https://techcrunch.com/feed/`, 재시도 최대 2회        |
| 3   | **주제 필터**               | Code                  | AI 키워드 매칭 → 통과 항목 중 **최신 1건** 선택, 제목·링크·본문·발행일·태그 추출 및 정규화 | 0건이면 `found:false` 반환                            |
| 4   | **뉴스 있음?**              | IF                    | 수집/필터 결과가 0건인지 분기                                          | `{{$json.found}} == true`                        |
| 5   | **알림: 뉴스 없음**           | NoOp                  | "오늘 해당 뉴스 없음" 처리(정상 종료). Slack/Email로 교체 가능                | —                                                |
| 6   | **Notion 중복 조회**        | HTTP Request → Notion | 원문 링크로 기존 페이지를 조회하여 **중복 여부 판단**                           | `POST /v1/databases/{id}/query`, `원문 링크 == link` |
| 7   | **신규 기사?**              | IF                    | 조회 결과 0건이면 신규로 분기                                          | `{{$json.results.length}} == 0`                  |
| 8   | **중복: 스킵**              | NoOp                  | 이미 저장된 기사 → 요약/저장 생략(멱등성·비용 0)                             | —                                                |
| 9   | **OpenAI 요약**           | HTTP Request → OpenAI | 본문을 **3줄 이내**로 요약 + 감성 산출 (기사당 1회 호출)                      | `gpt-4o-mini`, temp 0.3, max_tokens 300          |
| 10  | **요약 파싱**               | Code                  | 응답에서 3줄 요약과 `감성` 분리                                        | 불릿 라인 추출                                         |
| 11  | **Notion 저장**           | HTTP Request → Notion | 결과를 DB에 **속성 매핑**으로 저장                                     | `POST /v1/pages`                                 |
| 12  | **완료**                  | NoOp                  | 정상 종료 지점                                                   | —                                                |
| 13  | **에러 트리거**              | Error Trigger         | 실행 중 미복구 오류 발생 시 발화                                        | 워크플로우 Error Workflow=자기 자신                       |
| 14  | **알림: 처리 실패**           | NoOp                  | 운영자 알림(로그/스킵 후 통지). Slack/Email로 교체                        | —                                                |

## 3. 데이터 흐름 (필드 매핑)

| Notion 속성 | 타입           | 소스 표현식                                 |
| --------- | ------------ | -------------------------------------- |
| `제목`      | Title        | `{{ $('주제 필터').item.json.title }}`     |
| `요약`      | Rich text    | `{{ $('요약 파싱').item.json.summary }}`   |
| `원문 링크`   | URL          | `{{ $('주제 필터').item.json.link }}`      |
| `발행일시`    | Date         | `{{ $('주제 필터').item.json.isoDate }}`   |
| `출처`      | Rich text    | `TechCrunch`                           |
| `주제 태그`   | Multi-select | 매칭된 키워드 태그 배열                          |
| `감성`      | Select       | `{{ $('요약 파싱').item.json.sentiment }}` |
| `상태`      | Select       | `저장됨`                                  |

> n8n의 **item linking** 덕분에 중간 노드(중복 조회·요약)를 거쳐도 `$('주제 필터')`로 원본 기사 데이터에 계속 접근할 수 있습니다.

## 4. 안정성 / 에러 처리 설계

| 계층     | 메커니즘                          | 노드                          |
| ------ | ----------------------------- | --------------------------- |
| 일시 장애  | **재시도 최대 2회**(총 3회 시도), 5초 간격 | RSS·OpenAI·Notion(2,6,9,11) |
| 빈 결과   | 에러 아닌 **정상 분기**               | 뉴스 있음?(4) → 알림(5)           |
| 중복     | 저장 전 조회로 **건너뛰기**             | 중복 조회(6) → 신규 기사?(7)        |
| 미복구 오류 | **에러 워크플로우 발화 + 알림**          | 에러 트리거(13) → 알림(14)         |
| 무한 루프  | 트리거는 스케줄 1회뿐, 자가 호출 없음        | 스케줄(1)                      |

## 5. 가져오기(Import) & 설정 절차

1. **n8n 접속** → 우상단 **⋯ → Import from File** → `workflow/news-summary.n8n.json` 선택.
2. **자격증명(Credentials) 연결** *(JSON에는 키가 없고 참조만 있음)*
   - `OpenAI account` (openAiApi): OpenAI API Key 입력 → `OpenAI 요약` 노드에 연결.
   - `Notion account` (notionApi): Notion Internal Integration Token 입력 → `Notion 중복 조회`·`Notion 저장` 노드에 연결.
   - Notion 인티그레이션을 **대상 DB 페이지에 Connect**(공유)해야 API 접근 가능.
3. **타임존 확인**: 워크플로우 **Settings → Timezone = Asia/Seoul** (JSON에 포함됨).
4. **에러 워크플로우 지정**: **Settings → Error Workflow = (본 워크플로우)** 로 설정해야 `에러 트리거`가 동작.
5. **수동 테스트**: `스케줄 트리거`에서 **Execute Workflow** → 각 노드 초록 체크 및 Notion DB 새 행 확인.
6. **활성화**: 우상단 **Active** 토글 ON → 매일 09:00 자동 실행.

> DB ID는 이미 실제 값(`99f64246-aa35-4575-abd7-73c223bb2fce`)으로 주입되어 있습니다. 다른 DB로 바꾸려면 두 Notion 노드의 URL/`database_id`만 교체하세요.

## 6. 스크린샷 캡처 가이드 (산출물용)

제출용으로 아래 4장을 권장합니다. **캡처 전 키/토큰이 보이지 않도록** 자격증명 상세창은 닫아 주세요.

1. **전체 캔버스** — 노드 연결 구조 전체가 보이도록.
2. **실행 성공 화면** — 노드에 초록 체크가 찍힌 1회 실행 결과.
3. **Notion DB** — 저장된 행(제목/요약/링크/발행일시) 목록.
4. **스케줄/활성화** — 스케줄 트리거 설정과 Active=ON 상태.

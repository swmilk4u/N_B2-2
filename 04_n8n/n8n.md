# PB 뉴스 요약 자동화 워크플로우

RSS로 최신 기술(AI) 뉴스를 자동 수집하고, 생성형 AI로 **3줄 요약**한 뒤, **Notion 데이터베이스**에 자동 저장하는 무코드 자동화 파이프라인입니다.

- **자동화 툴:** n8n
- **생성형 AI:** OpenAI `gpt-4o-mini`
- **저장소:** Notion
- **실행 주기:** 매일 09:00 (Asia/Seoul)

> 📌 Notion 산출물
> - 프로젝트 허브: https://app.notion.com/p/38cf7fb53c31815cb2a4da23660ac2ef
> - 뉴스 요약 DB: https://app.notion.com/p/99f64246aa354575abd773c223bb2fce

## 2. 파이프라인 한눈에 보기

```
스케줄(매일 09:00) → RSS 수집 → 주제 필터(AI, 최신 1건)
   → 중복 조회(원문 링크) → OpenAI 3줄 요약 → Notion 저장
   ↘ 뉴스 없음/중복 → 스킵      ↘ 오류 → 재시도(최대 2회) → 알림
```

## 4. 주제 필터링 기준과 선택 이유

### 4.2 태그 분류 규칙 (Notion `주제 태그`)

### 4.3 선택 이유

- **한/영 핵심어 혼용**으로 AI 도메인 누락을 최소화하고, 일반 기술 피드에서도 AI 주제만 정확히 추립니다.
- **최신 1건만 선택**: 과제 요구(1건 추출)와 비용 절감(요약 1회 호출) 원칙을 동시에 만족합니다.
- 태그를 함께 저장해 **"왜 이 기사가 선택됐는지" 근거를 데이터로 남깁니다.**

## 5. 에러 처리 정책과 선택 이유

| 상황 | 처리 | 선택 이유 |
|------|------|-----------|
| 수집/필터 **0건** | 에러가 아닌 **정상 분기**로 "뉴스 없음" 알림 후 종료 | 뉴스 부재는 실패가 아니므로 불필요한 에러·재시도·비용을 막음 |
| **일시적 실패**(타임아웃·5xx·레이트리밋) | **자동 재시도 최대 2회**(5초 간격) | 일시 장애는 재시도로 대부분 복구되며, 상한으로 비용 폭주를 차단 |
| **재시도 후에도 실패** | 로그 기록 + 스킵, **에러 트리거 → 운영자 알림** | 1건 실패가 전체 파이프라인을 멈추지 않도록 격리, 사후 인지는 가능 |
| **중복 기사** | 원문 링크 키로 조회 후 **저장/요약 생략** | 멱등성 보장, 중복 저장·중복 과금 방지 |
| **무한 루프** | 트리거는 스케줄 1회뿐, 자가 호출 경로 없음 | 과제 금지사항(무한 루프·불필요 호출) 준수 |

- **중복 방지 키:** `원문 링크`(URL) — 추적 파라미터 제거 후 정규화하여 동일 기사 변형을 흡수.
- **요약 비용 원칙:** 중복 체크를 **요약보다 먼저** 수행 → 이미 저장된 기사엔 OpenAI를 호출하지 않음.

## 6. 빠른 시작

1. n8n에서 [`workflow/news-summary.n8n.json`](workflow/news-summary.n8n.json) **Import from File**.
2. `OpenAI account`(openAiApi), `Notion account`(notionApi) 자격증명 생성 후 각 노드에 연결.
   - Notion 인티그레이션을 **뉴스 요약 DB 페이지에 Connect**(공유).
3. **Settings → Error Workflow = 본 워크플로우**, **Timezone = Asia/Seoul** 확인.
4. **Execute Workflow**로 수동 테스트 → Notion DB에 새 행 생성 확인.
5. **Active** 토글 ON → 매일 09:00 자동 실행.

> 상세 절차·스크린샷 가이드: [docs/workflow-design.md](docs/workflow-design.md)

## 7. 보안

- API 키/토큰은 **n8n Credentials(암호화 저장)** 로만 관리하며, 워크플로우 JSON에는 **참조 ID만** 포함됩니다(평문 키 없음).
- 로컬 보조 변수는 `.env`로 분리하고 `.gitignore`로 추적 제외합니다. 저장소에는 마스킹본 `.env.example`만 커밋합니다.
- 문서·스크린샷에는 키/토큰을 `sk-****`, `secret_****` 형태로 마스킹합니다.

## 8. 보너스 과제 (선택)

- **썸네일 자동 생성:** OpenAI 이미지 생성으로 기사 주제 썸네일을 만들어 `썸네일` 속성에 첨부 (DB 속성 준비 완료).
- **감성 분석 태그:** 요약과 **동일 응답**에서 `sentiment(긍정/중립/부정)`을 산출해 `감성` 속성에 매핑 (추가 호출 없음).
